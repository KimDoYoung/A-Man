# 인증

  현재 A-Man 시스템의 로그인(인증) 프로세스는 JWT(JSON Web Token) 및 쿠키(Cookie) 기반 인증 방식을
  결합하여 구현되어 있으며, 구체적인 동작 흐름은 다음과 같습니다.
  ──────
  ### 1. 프론트엔드 로그인 요청

  1. 사용자가  LoginPage.tsx 에서 **아이디(Username)**와 **비밀번호(Password)**를 입력하고 로그인
  버튼을 클릭합니다.
  2. 프론트엔드는  /aman/auth/login  엔드포인트로 JSON 바디에 담아 POST 요청을 전송합니다.
  ──────
  ### 2. 백엔드 인증 및 토큰 발급 ( AuthController.java )

  1. 사용자 조회 및 활성 상태 체크:
      • 입력받은 아이디를 기반으로 DB( users  테이블)에서 사용자 정보를 조회합니다.
      • 이때 계정 활성화 필드인 ** is_active = 1 **인 사용자만 정상 로그인 대상으로 판단합니다.
  2. 비밀번호 비교:
      • 현재 개발 요건이 반영되어 비밀번호는 단방향 해시 암호화가 아닌 평문 비밀번호 문자열 비교
      방식으로 매칭 여부를 검사합니다. ( password.equals(user.getPassword()) )
  3. JWT 토큰 생성:
      • 인증이 성공하면  JwtTokenProvider.java 를 통해 두 가지 토큰을 만료 시간과 함께 신규
      발급합니다.
          • Access Token: 권한 정보가 포함된 1시간 만료 토큰.
	  • Refresh Token: 세션 유지를 위한 14일 만료 토큰.

  4. Refresh Token DB 저장 및 이전 토큰 삭제:
      • 동일한 사용자의 이전 Refresh Token 데이터가 있을 시 중복 로그인을 방지하기 위해 이를 파기하고
      새로운 Refresh Token 정보를  refresh_tokens  테이블에 저장합니다.
  5. HttpOnly 쿠키 설정:
      • XSS(Cross Site Scripting) 공격 방어와 보안을 강화하기 위해 생성된  access_token 과
      refresh_token 을  HttpOnly  속성의 브라우저 쿠키로 강제 탑재하여 브라우저로 응답을 내려보냅니다.
  6. 응답 전송:
      • 응답 바디에는 사용자의 식별 정보(ID, 아이디, 역할, 이메일, 이름)와  accessToken 을 JSON
      형태로 반환합니다.

  ──────
  ### 3. 클라이언트의 인증 상태 관리 ( App.tsx )

  1. 로컬 스토리지 저장:
      • 로그인 성공 응답을 수신하면 사용자 기본 메타 데이터( response.data )를 브라우저의
      localStorage 에  'aman_user' 라는 키로 저장합니다.
  2. API 호출 시 인증 헤더 자동 탑재:
      • Axios Request Interceptor가 설정되어 있어, 웹 애플리케이션의 모든 API 요청 전송 시 로컬
      스토리지의 토큰 정보가 존재할 경우 요청 헤더에 자동으로 ** Authorization: Bearer [accessToken]
      **를 주입하여 백엔드로 요청을 전송합니다.
  3. 인증 만료 감지 및 리다이렉트:
      • Axios Response Interceptor는 백엔드로부터  401(Unauthorized)  또는  403(Forbidden)  응답을
      받게 되면, 로컬 스토리지 세션을 파기하고 사용자에게 "로그인 세션 만료" 안내와 함께 자동으로
      /login  페이지로 리다이렉트 처리합니다.

  ──────
  ### 4. 로그아웃 처리

  1. 로그아웃 API 요청 시 백엔드는 DB의  refresh_tokens 에서 해당 토큰을 삭제하여 만료시킵니다.
  2. 보안 상의 이유로 아직 유효기간이 남아 있는 Access Token에 의한 오용을 막기 위해 로그아웃 시점의
  Access Token을  token_blacklist  테이블에 등록하여 유효 시간이 지날 때까지 영구 차단합니다.
  3. 브라우저의  access_token  및  refresh_token  쿠키 정보를 파기(MaxAge = 0)합니다.
