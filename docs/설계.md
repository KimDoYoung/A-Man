
# AMan 시스템 설계

## 개요

- AssetERP의 매뉴얼을 작성하고 이를 사용자들에게 제공한다.
- AssetERP는 한국펀드서비스가 자산운용사에 제공하는 ERP시스템이다.
- 매뉴얼 페이지를 markdown으로 sqlite에 저장한다.
- 일반사용자들에게 AssetERP의 매뉴얼 사이트를 제공한다.

## domain 용어

- 일반사용자 : AssetERP 서비스를 제공받는 자산운용회사의 사원들
- 문서사용자 : 한국펀드서비스에서 AssetERP 도움말 페이지를 작성하는 사원들
- 매뉴얼 사이트 : https://aview.k-fs.co.kr:7171/aman

## 기능

1. 일반사용자들에게 매뉴얼 사이트를 제공한다.
2. 매뉴얼사이트는 AssetERP의 모든 기능에 대한 도움말 페이지를 제공한다.
3. AssetERP는 3 depth의 메뉴시스템을 갖고 있음. 각 페이지마다 도움말 페이지가 필요함.
4. 왼쪽에 트리 3단계가 있고 AssetERP의 메뉴체계를 그대로 가져가면서 각 메뉴마다 도움말 페이지가 존재하는 구조여야함.
5. /admin 폴더  하위는 모두 jwt로 인증되어야한다.
6. 문서사용자가 markdown으로 작성한 내용은 sqlite에 저장된다.

## 필수 사항

1. java 1.8을 사용
2. tomcat 8.5에 war로 배포함.
3. application.properties를 사용.  **yml사용하지 않음**
4. client(react)에서 markdown->html로 변경함
5. 환경변수 AMAN_MODE에 따라서 application-<AMON_MODE>.properties
6. BASE_DIR 을 기준으로 하위 폴더 db,logs,data/images 등을 작성한다.
