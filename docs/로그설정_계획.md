# 로그 설정 개선 계획서 (Profile-driven Logging Plan)

이 계획서는 서버(test 프로필) 배포 환경에서 불필요한 `DEBUG` 및 `TRACE` 수준의 SQL 로그가 대량으로 출력되는 현상을 해결하기 위해, 로그 설정을 스프링 프로필(Profile)별로 동적 제어하도록 수정하는 방안을 설명합니다.

---

## 1. 개요 및 원인 분석

1. **Logback 설정 내 레벨 하드코딩**: 
   - `logback-spring.xml` 내에 패키지(`kr.co.kfs.aman`) 레벨이 `DEBUG`로, Hibernate SQL 로거(`org.hibernate.SQL`)가 `DEBUG`로, 파라미터 바인더(`org.hibernate.type.descriptor.sql.BasicBinder`)가 `TRACE`로 **강제 지정(하드코딩)**되어 있었습니다.
   - Spring Boot의 로그 설정 우선순위상, Logback 파일 내에 명시된 하드코딩 값이 `application.properties` 설정보다 우선하므로 `application-test.properties`의 설정들이 전부 무시되고 있었습니다.
2. **테스트 프로필의 SQL 노출 옵션 활성화**:
   - `application-test.properties` 파일에 `spring.jpa.show-sql=true` 옵션이 켜져 있어 Hibernate가 표준 콘솔 스트림으로 모든 쿼리를 출력하고 있었습니다.

---

## 2. 해결 방안

1. **`logback-spring.xml` 동적 바인딩**:
   - Logback 설정 내 하드코딩된 레벨값들을 Spring Environment 프로퍼티(`logging.level.*`)와 연동되는 `<springProperty>`로 변경합니다.
   - 이를 통해 각 프로필 설정 파일(`application-local.properties`, `application-test.properties`)의 로그 레벨 설정이 Logback에 실시간 상속됩니다.
2. **환경별 로그 레벨 제어**:
   - **로컬 개발 환경 (`local`)**: 개발 편의를 위해 `DEBUG` 레벨 및 SQL 쿼리 상세 출력(`TRACE`)을 유지합니다.
   - **배포/테스트 환경 (`test`)**: 로그 간소화 및 성능 최적화를 위해 패키지 및 SQL 출력 수준을 `INFO`로 변경하고 `spring.jpa.show-sql`을 `false`로 비활성화합니다.

---

## 3. 수정 대상 파일 및 변경 내용

### 3.1 백엔드 설정 컴포넌트

#### [MODIFY] [logback-spring.xml](file:///home/kdy987/work/aman/backend/src/main/resources/logback-spring.xml)
- 스프링 프로퍼티 연동을 위한 `<springProperty>` 정의 추가:
  ```xml
  <springProperty name="LOG_LEVEL_AMAN" source="logging.level.kr.co.kfs.aman" defaultValue="INFO"/>
  <springProperty name="LOG_LEVEL_SQL" source="logging.level.org.hibernate.SQL" defaultValue="INFO"/>
  <springProperty name="LOG_LEVEL_BINDER" source="logging.level.org.hibernate.type.descriptor.sql.BasicBinder" defaultValue="INFO"/>
  ```
- 기존 하드코딩된 레벨값을 변수로 대체:
  ```xml
  <logger name="kr.co.kfs.aman" level="${LOG_LEVEL_AMAN}" additivity="false"> ... </logger>
  <logger name="org.hibernate.SQL" level="${LOG_LEVEL_SQL}" additivity="false"> ... </logger>
  <logger name="org.hibernate.type.descriptor.sql.BasicBinder" level="${LOG_LEVEL_BINDER}" additivity="false"> ... </logger>
  ```

#### [MODIFY] [application-test.properties](file:///home/kdy987/work/aman/backend/src/main/resources/application-test.properties)
- SQL 출력 옵션 비활성화 및 로거 수준을 `INFO`로 제한:
  ```properties
  spring.jpa.show-sql=false
  logging.level.kr.co.kfs.aman=INFO
  logging.level.org.hibernate.SQL=INFO
  logging.level.org.hibernate.type.descriptor.sql.BasicBinder=INFO
  ```

#### [MODIFY] [application-local.properties](file:///home/kdy987/work/aman/backend/src/main/resources/application-local.properties)
- 로컬 개발 환경용 로그 상세 출력 설정 유지/정교화:
  ```properties
  logging.level.kr.co.kfs.aman=DEBUG
  logging.level.org.hibernate.SQL=DEBUG
  logging.level.org.hibernate.type.descriptor.sql.BasicBinder=TRACE
  ```
