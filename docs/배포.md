# 배포

## jskn 서버에 배포

### 상황

1. jskn은 docker에 여러 app을 올려 놓고 서비스한다.
2. nginx로 reverse proxy를 사용하고 있다.
3. jskn서버의 docs/docker-compose.yml과 default.conf 을 확인하기 바람.
4. A-Man app은 자바 1.8 임으로 jskn의 tomcat을 사용할 수 없다.
5. A-Man의 자체 image를 만들어서 따로 docker에 올려야 한다.
6. 기존 aview 서비스가 자바1.8로 만들어져서 비슷한 상황이다.
7. jskn 파일 위치
    /data/docker/docker-compose.yml
    /data/docker/nginx/conf.d/default.conf
8. 내가 jskn서버에 /data/docker/apps/aman/db 을 생성하고 aman.db를 생성해 둠

### 방법

1. jskn 서버 : /data/docker/apps/aman/data 폴더를 생성해 둠 
2. Dockerfile을 생성
3. deploy-jskn.sh 에서 aman.war를 만듬
4. deploy-jskn.sh 에서 /data/docker/apps/aman/data으로  아래 2개의 파일 업로드
    1. aman.war 
    2. Dockerfile 
5. jskn 서버의 docker-compose.yml 
```yml
  aman:
    build:
      context: /data/docker/apps/aman/data   # Dockerfile과 aview.war가 있는 경로
      dockerfile: Dockerfile
    container_name: jskn_aman
    user: "1000:1000" # kdy987
    restart: always
    # Nginx가 내부망(8080)으로 통신하므로 외부 포트는 열지 않아도 되지만, 
    # 초기 테스트용으로 8081 등을 열어둘 수 있습니다.
    ports:
      - "8686:8080"
    volumes:
      - /data/docker/apps/aman:/data/docker/apps/aman:Z
    environment:
      - TZ=Asia/Seoul
      - AMAN_MODE=jskn
      - SPRING_PROFILES_ACTIVE=jskn
```
6. jskn서버의 default.conf 수정

```
    location /aman {
        proxy_pass http://aman:8080/aman; # 도커 네트워크 브릿지를
  통해 컨테이너 명으로 통신
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
```
7. jskn  docker 재기동
8. nginx 재기동

## jskn에 배포2

  ### 1.  docs/default.conf  (Nginx 설정) 수정

  Nginx가 외부의  /aman/  요청을 새로 띄울  jskn_aman  도커 컨테이너로 리버스 프록시(Reverse Proxy) 하도록  location
  /aman/  블록을 추가합니다. ( location /aview/  아래쪽에 추가하시면 좋습니다.)

        # --- aman (AssetERP 매뉴얼 시스템) 설정 추가 ---
        location /aman/ {
            proxy_pass http://jskn_aman:8080/aman/;

            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;

            # 대용량 파일 및 마크다운 업로드를 위해 제한 용량 및 타임아웃 상향
            client_max_body_size 50M;
            proxy_read_timeout 120s;
        }
    ──────
  ### 2.  docs/docker-compose.yml  수정

   services:  항목 하위에  aman  서비스 정의를 추가합니다.  deploy-jskn.sh 가 전송해 준  /data/docker/apps/aman/data
  위치의  Dockerfile 과  aman.war 를 기반으로 빌드하도록 설정합니다.

      # 11. aman 앱 (AssetERP 매뉴얼 시스템)
      aman:
        build:
          context: /data/docker/apps/aman/data   # deploy-jskn.sh가 전송한 경로
          dockerfile: Dockerfile
        image: jskn_aman_img
        container_name: jskn_aman
        user: "1000:1000"
        restart: always
        ports:
          - "8686:8080"  # (선택 사항) 외부 직접 접속 테스트용 포트
        volumes:
          - /data/docker/apps/aman:/data/docker/apps/aman  # DB(aman.db), 로그, 이미지 데이터 보존용 볼륨
        environment:
          - TZ=Asia/Seoul
          - AMAN_MODE=jskn  # application-jskn.properties 설정을 로드하기 위한 환경변수

  (추가로  nginx  서비스의  depends_on:  목록 하위에  - aman 을 추가해 두면 좋습니다.)
  ──────
  ### 3. jskn 서버에서 실행해야 하는 명령어

  파일 전송 후 jskn 서버에서 도커 이미지를 빌드하고 컨테이너를 띄워야 최종 배포가 완료됩니다.

  #### 로컬 PC에서 배포 파일 전송

    ./deploy-jskn.sh
    # (만약 DB까지 새로 초기화하려면 ./deploy-jskn.sh --init-db)

  #### jskn 서버 SSH 접속

    ssh -p 2020 kdy987@jskn.iptime.org

  #### jskn 서버에서 도커 빌드 및 컨테이너 실행

   docker-compose.yml  파일이 위치한 디렉터리(예:  /data/docker )로 이동하여 아래 명령어를 수행합니다.

    cd /data/docker

    # 1. 수정된 default.conf가 nginx 설정 경로에 반영되어 있는지 확인/복사
    # cp /경로/default.conf /data/docker/nginx/conf.d/default.conf

    # 2. aman 컨테이너 빌드 및 nginx와 함께 재시작
    docker-compose up -d --build aman nginx

  #### 접속 확인

  위 명령어가 정상 실행된 후 브라우저에서 https://jskn.iptime.org/aman 으로 접속하면 정상적으로 서비스가 노출됩니다.
