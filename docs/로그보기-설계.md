# A-Man 로그 조회 및 관리 시스템 설계서 (로그보기-설계.md)

이 문서는 A-Man 프로젝트의 시스템 로그(실시간 로그 조회, 보관 로그 파일 다운로드 및 삭제, 스프링 로그 제어)에 관한 전체 백엔드 API 설계 및 프론트엔드 UI/UX 사양을 종합하여 하나로 정리한 통합 설계 문서입니다.

---

## 1. 개요 및 요구사항 (Overview & Requirements)

1. **실시간 로그 모니터링 (Tail & Scroll)**
   - WAS의 실시간 로그 파일(`A-Man.log` 또는 `aman.log`) 내용을 효율적으로 슬라이싱하여 웹 단독 콘솔 터미널 화면에 노출합니다.
   - 대용량 파일(10MB 이상)을 한 번에 읽지 않고, 지정된 최신 N줄(100/200/500/1000)만 슬라이싱 조회 및 자동 스크롤(Auto Scroll to bottom)을 지원합니다.
2. **보관 로그 파일 관리 (Download & Clean)**
   - 롤오버되어 압축된 백업 로그 파일들(`*.log.gz`)의 목록을 조회하고 브라우저를 통해 안전하게 다운로드합니다.
   - 오래되어 용량을 차지하는 압축 로그 파일(`.gz`)에 대해 휴지통 삭제 기능을 제공합니다.
3. **환경별 로그 레벨 제어 및 호환성**
   - 로컬 개발 환경(`local`)과 배포/운영 환경(`test`) 간 로그 설정 파일을 스프링 프로필(Profile)과 연동하여 동적으로 전환합니다.
   - 파일명 불일치(로컬: `A-Man.log`, 배포지: `aman.log`) 문제를 순차 검색 폴백을 통해 투명하게 처리합니다.
4. **보안 및 예외 차단**
   - 파일 다운로드 및 삭제 시 상위 디렉토리 우회 공격(`Path Traversal`)을 방지합니다.
   - 현재 작동 중인 실시간 로그 파일(`A-Man.log`)은 절대 삭제할 수 없도록 백엔드와 프론트엔드 모두에서 안전장치를 둡니다.

## 2. 백엔드 설계 및 구현 (Backend Implementation)

### 2.1 API 엔드포인트 사양 ([LogController.java](file:///home/kdy987/work/aman/backend/src/main/java/kr/co/kfs/aman/controller/LogController.java))
백엔드는 `/admin/logs` 하위 경로에 인증 필터(JWT)를 거쳐 동작하는 컨트롤러를 탑재합니다.

- **`GET /admin/logs/files`**: 로그 폴더 내 파일 목록 리스트 및 절대 경로 조회 (응답 예: `{"files": [...], "logDir": "/data/..."}`)
- **`GET /admin/logs/tail?lines=N`**: 실시간 로그 파일의 최신 N줄 내용 조회 (UTF-8, 250KB 역방향 메모리 탐색)
- **`GET /admin/logs/download/{fileName}`**: 지정한 로그 파일 다운로드 (GZIP 스트림 호환)
- **`DELETE /admin/logs/delete/{fileName}`**: 지정한 압축 보관 로그 파일 삭제 (안전성 검사 포함)

### 2.2 실시간 로그 파일 검색 폴백 (Fallback Logic)
스프링 어플리케이션 이름 주입 및 물리적 존재 여부 순차 체크 방식을 통해 로컬/서버 편차를 극복합니다:
```java
private File getActiveLogFile(File logDir) {
    File f1 = new File(logDir, appName + ".log");
    if (f1.exists()) {
        return f1;
    }
    File f2 = new File(logDir, "aman.log");
    if (f2.exists()) {
        return f2;
    }
    File f3 = new File(logDir, "A-Man.log");
    if (f3.exists()) {
        return f3;
    }
    return f1;
}
```

### 2.3 프로필 기반 Logback 및 JPA 로깅 제어
`logback-spring.xml` 내의 하드코딩 레벨 선언을 스프링 프로퍼티와 동적으로 연결하고 환경 프로파일에 따라 선택 상속합니다.
- **`logback-spring.xml` 설정**:
  ```xml
  <springProperty name="LOG_LEVEL_AMAN" source="logging.level.kr.co.kfs.aman" defaultValue="INFO"/>
  <springProperty name="LOG_LEVEL_SQL" source="logging.level.org.hibernate.SQL" defaultValue="INFO"/>
  <springProperty name="LOG_LEVEL_BINDER" source="logging.level.org.hibernate.type.descriptor.sql.BasicBinder" defaultValue="INFO"/>
  ```
- **테스트/배포 프로필 (`application-test.properties`)**: SQL 노출을 끄고 로그 용량을 줄이기 위해 `INFO` 레벨 지정.
- **로컬 개발 프로필 (`application-local.properties`)**: 쿼리 확인 및 디버깅을 위해 `DEBUG` 및 파라미터 바인더 `TRACE` 유지.

### 2.4 안전한 로그 파일 삭제 검증 로직
경로 우회 문자를 검출하고, 현재 구동 로그의 임의 삭제를 차단하기 위해 **오직 `.gz`로 끝나는 파일만 삭제할 수 있도록 제한**합니다:
```java
if (fileName.contains("..") || fileName.contains("/") || fileName.contains("\\")) {
    return ResponseEntity.badRequest().body("잘못된 로그 파일명입니다.");
}
if (!fileName.endsWith(".gz")) {
    return ResponseEntity.badRequest().body("보관된 압축 로그 파일(.gz)만 삭제할 수 있습니다.");
}
```

## 3. 프론트엔드 설계 및 UI/UX 개선 (Frontend Implementation)

### 3.1 가독성 중심 테마 (미색 & 에스프레소)
장시간 시스템 장애 추적 시 눈의 피로를 예방하기 위해, 전자책 및 고급 서적의 Sepia/e-book 스타일을 콘솔 터미널 테마에 이식합니다.
- **배경색**: 따뜻한 미색 (`#fdfaf2`)
- **기본 텍스트**: 다크 에스프레소 브라운 (`#4e342e`)
- **로그 등급별 고대비 색상**:
  - `ERROR` / `Exception`: 붉은색 굵은 텍스트 (`text-red-700 font-bold`)
  - `WARN`: 딥 앰버 (`text-amber-800 font-semibold`)
  - `INFO`: 포레스트 그린 (`text-emerald-700 font-medium`)
  - `DEBUG`: 미디엄 그레이 (`text-slate-500`)
  - `TRACE`: 라이트 그레이 (`text-slate-400`)
- **텍스트 크기**: 기존 `11px`에서 1pt 높은 **`12px` (`text-[12px]`)**로 상향 적용하여 장독 가독성 최적화.
- **로그 저장 폴더 경로 노출**: 콘솔 상단 설명 우측에 `로그폴더: {logDir}` 뱃지(배경 `bg-slate-100`, 테두리 `border-slate-200`)를 렌더링하여 관리자의 파일 디렉토리 편의성을 도모합니다.
- **하단 통계바 및 노이즈 제거**: 터미널 최하단에 존재하던 불필요한 구분선 및 `"총 N개 라인 표시됨"` 텍스트 영역을 완전 삭제하여 미니멀하고 몰입감 있는 레이아웃 완성.

### 3.2 접이식 보관 로그 사이드바 토글 및 목록 제한
- **목록 최적화**: 보관된 파일이 많아 스크롤이 지연되는 것을 막기 위해 최신순 상위 **최대 10개**만 렌더링을 제한하고, 제목에 `보관 로그 파일 (최근 10개)`로 동적 노출합니다.
- **레이아웃 접기 토글**: 콘솔 툴바 영역에 `보관로그 숨기기/보이기` 버튼 및 눈 모양(`Eye`, `EyeOff`) 아이콘을 추가하여 사이드바 열림 상태를 배타적으로 접어 콘솔 화면을 넓게 쓰도록 지원합니다.

### 3.3 검색 키워드 실시간 하이라이트 및 X 지우기 버튼
- **전체 맥락 보존**: 검색창에 단어를 기입하더라도 비매칭 행을 가리지 않고 **로그 전체 내용을 보존**하되, 검색 단어만 빨간색 하이라이트(`bg-red-100 text-red-700 font-bold px-0.5 rounded`)로 실시간 마킹 처리해 SQL 쿼리의 흐름을 깨뜨리지 않습니다.
- **원클릭 지우기**: 입력 필드 내 우측 끝에 `X` 버튼을 절대 배치(`absolute right-2.5`)하여 한 번에 검색어를 클리어하도록 지원하며, input 우측 패딩을 `pr-8`로 주어 텍스트 겹침을 방지합니다.

### 3.4 휴지통 아이콘 삭제 제어 및 다이나믹 배치
활성 로그와 보관 압축 로그의 중요성에 따라 카드의 레이아웃을 다르게 처리합니다.
- **활성 구동 로그 (`file.name.endsWith('.log')`)**:
  - `isActiveLog`가 참이 되며 다운로드 버튼만 노출되고, 글자 가림 패딩은 `pr-8`로 콤팩트하게 적용합니다.
- **과거 압축 로그 (`file.name.endsWith('.gz')`)**:
  - 우측 상단에 다운로드(`right-9`) 및 휴지통 삭제(`right-2.5` `Trash2`) 버튼이 병렬 배치되며, 글자 겹침 방지를 위해 텍스트 패딩을 `pr-14`로 확보합니다.
  - 삭제를 누르면 `confirm` 모달 확인 과정을 거쳐 백엔드 API를 호출한 후 목록을 갱신합니다.

### 3.5 Vite 개발용 프록시 F5(새로고침) 라우팅 우회
- **문제점**: Vite 개발 서버(port 5173)에서 `/aman/admin/logs`, `/aman/admin/backup` 등 프론트엔드 라우팅 경로를 F5로 새로고침하면, `vite.config.ts`의 백엔드 프록시 매칭 정규식(`admin/(logs|backup)...`)에 걸려 백엔드로 포워딩됩니다. 백엔드에는 이에 대응하는 HTML 컨트롤러가 없으므로 빈 화면이 렌더링되는 오류가 발생합니다.
- **해결 방안**: 프록시 옵션에 `bypass` 함수를 추가하여, 브라우저의 일반 페이지 로드 요청인 경우(요청 헤더 `accept`에 `html`이 포함된 경우) 프록시를 중단하고 base 경로가 매핑된 `'/aman/index.html'`을 명시적으로 반환함으로써 Vite가 해당 진입점 자원을 올바르게 렌더링하고 React Router가 서브 경로를 온전히 읽어들여 튕김 없이 같은 페이지를 표시하도록 설계합니다.
