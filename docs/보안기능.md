관리자분의 “매뉴얼도 보안이다”라는 철벽 방어 태세에 많이 당황하셨겠네요! 일반 사용자용 서비스인데 인쇄와 소스보기까지 막아달라니 개발자 입장에서는 한숨이 나올 만한 상황입니다.

결론부터 말씀드리면, **웹 브라우저 구조상 클라이언트(사용자 화면)에 한 번 뜬 데이터는 100% 완벽하게 캡처나 복사를 막을 수 없습니다.** (F12 개발자 도구만 열어도 다 보이니까요.)

하지만 관리자분을 안심시키고 "우리도 할 수 있는 조치는 다 했다"라고 보여주기 위한 화면 뷰어 수준의 방어 조치(기타 등등)를 정리해 드립니다.

## 1. 요청하신 핵심 기능 구현 방법

### 1) 인쇄(프린트/PDF 저장) 방지

CSS의 미디어 쿼리(`@media print`)를 사용해 인쇄 화면이 호출되면 전체 콘텐츠를 숨기거나 깨지게 만듭니다.

```css
@media print {
    /* 인쇄 시 화면 전체를 하얗게 숨김 */
    body {
        display: none !important;
    }
}

```

> 💡 **한계:** 브라우저 인쇄 창(Ctrl + P)은 막을 수 있지만, OS 수준의 전체 화면 캡처(Print Screen, Win + Shift + S)는 막지 못합니다.

### 2) 우클릭 및 키보드 단축키(소스보기, F12) 차단

JavaScript를 활용해 마우스 우클릭 메뉴를 막고, 개발자 도구를 열 수 있는 주요 단축키를 차단합니다. Fish Shell이나 현대적인 웹 프레임워크 어디서든 Script 단에 심을 수 있습니다.

```javascript
// 1. 우클릭 방지
document.addEventListener('contextmenu', event => event.preventDefault());

// 2. 주요 단축키 방지 (F12, Ctrl+Shift+I, Ctrl+Shift+J, Ctrl+U 등)
document.addEventListener('keydown', error => {
    if (error.keyCode === 123) { // F12
        error.preventDefault();
    }
    if (error.ctrlKey && error.shiftKey && (error.keyCode === 73 || error.keyCode === 74)) { // Ctrl+Shift+I / J
        error.preventDefault();
    }
    if (error.ctrlKey && error.keyCode === 85) { // Ctrl+U (소스 보기)
        error.preventDefault();
    }
});

```

> 💡 **한계:** 브라우저 설정에서 JavaScript를 끄고 접속하거나, 주소창 왼쪽에 `view-source:`를 강제로 붙이면 소스가 보입니다.

## 2. 관리자를 만족시킬 수 있는 "기타 등등" 보안 아이디어

단순 스크립트 차단 외에, "이 정도로 보안을 신경 쓰고 있다"라고 보고할 만한 추가 기술적 조치들입니다.

### 3) 텍스트 드래그 및 복사 금지 (CSS)

사용자가 매뉴얼 내용을 드래그해서 메모장에 복사-붙여넣기 하는 것을 막습니다.

```css
body {
    -webkit-user-select: none; /* Safari */
    -ms-user-select: none;     /* IE 10+ */
    user-select: none;         /* Standard */
}

```

### 4) 화면 워터마크(Watermark) 도입

가장 현실적이고 강력한 심리적 보안 대책입니다. 배경에 **'접속자 IP'** 또는 **'현재 시간'**, 아니면 **'A-Man Confidential'** 같은 문구를 반투명하게 깔아둡니다.

* **효과:** 사용자가 폰으로 화면을 직접 촬영하거나 캡처해서 유출하더라도, 누구의 계정/IP에서 유출되었는지 추적이 가능하므로 유출 의욕을 꺾어버립니다.

### 5) 텍스트가 아닌 이미지/PDF 뷰어 형태로 렌더링

HTML 텍스트로 매뉴얼을 띄우면 긁어가기가 너무 쉽습니다.

* 매뉴얼을 이미지(PNG) 파일로 변환해서 보여주거나, `pdf.js` 같은 라이브러리를 이용해 웹 뷰어 형태로 렌더링합니다. 이러면 드래그나 소스보기로 텍스트를 그대로 가져가는 것이 불가능해집니다.

### 6) 개발자 도구(F12) 오픈 시 무한 디버거(Debugger) 락

단축키를 뚫고 F12를 강제로 연 악성(?) 사용자를 엿먹이는(?) 기법입니다. 소스코드에 주기적으로 `debugger;` 코드가 실행되도록 루프를 돌려버리면, 개발자 도구가 켜지는 순간 브라우저가 멈추거나 디버그 모드에 갇혀 소스를 제대로 볼 수 없게 됩니다.

관리자분에게 이 기능을 적용해 주면서 아래와 같이 보고하시면 깔끔합니다.

> "요청하신 **우클릭 차단, 소스보기 단축키 제한, 인쇄 화면 숨김, 텍스트 복사 금지** 조치를 모두 적용했습니다.
> 다만, 기술적으로 브라우저 기능이나 스마트폰 촬영까지 완벽히 막는 것은 불가능하므로, 추후 보안 레벨을 더 높여야 한다면 **'화면 워터마크 도입'**이나 **'이미지 뷰어 방식'**으로 고도화하는 방향을 추천해 드립니다."

웹 보안의 세계는 막으려는 자와 뚫으려는 자의 창과 방패 싸움이지만, 이 정도 '겉포장'만 해두어도 일반 사용자 대상으로는 충분한 방어벽이 됩니다. 화이팅입니다!
