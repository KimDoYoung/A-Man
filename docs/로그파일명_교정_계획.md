# 로그 파일명 조회 교정 계획서 (Fix Log File Name Resolution Plan)

이 계획서는 로그 조회 화면 진입 시 "로그 기록이 비어있거나 파일이 존재하지 않습니다." 문구가 뜨는 현상을 해결하기 위해, 로컬 개발 환경(`A-Man.log`)과 운영 배포 환경(`aman.log`) 모두에서 실시간 로그 파일을 안전하게 찾도록 순차 검색 폴백(Fallback) 방식을 구현하는 방안을 설명합니다.

---

## 1. 원인 분석
- `logback-spring.xml`에서 로그 파일명으로 `${APP_NAME}.log`를 설정하고 있으며, 이 값은 스프링 프로퍼티 `spring.application.name`에서 읽어옵니다.
- **로컬 개발 환경**: `spring.application.name=A-Man`이므로 **`A-Man.log`**가 생성됩니다.
- **배포 서버 환경**: 서버 기동 구성이나 패키징에 따라 **`aman.log`** 파일로 로그가 기록됩니다.
- 기존 `LogController`는 `"aman.log"`로 하드코딩되어 있어 로컬 환경(`A-Man.log`)에서 동작할 때 파일을 찾지 못해 오류가 발생했습니다.

---

## 2. 해결 방안
- `LogController`가 특정 고정 파일명만 찾는 대신, 순차적으로 파일을 검사하는 `getActiveLogFile` 폴백 로직을 작성합니다:
  1. `appName.log` (예: `A-Man.log`)가 존재하면 사용
  2. `aman.log`가 존재하면 사용
  3. `A-Man.log`가 존재하면 사용
  4. 모두 없으면 기본값인 `appName.log` 반환

---

## 3. 수정 대상 파일 및 변경 내용

### [Component: Backend Controller]

#### [MODIFY] [LogController.java](file:///home/kdy987/work/aman/backend/src/main/java/kr/co/kfs/aman/controller/LogController.java)
- 클래스 상단에 어플리케이션 이름 주입:
  ```java
  @Value("${spring.application.name:aman}")
  private String appName;
  ```
- 순차 검색 폴백 메소드 추가 및 적용:
  ```java
  private File getActiveLogFile(File logDir) {
      File f1 = new File(logDir, appName + ".log");
      if (f1.exists()) {
          return f1;
      }
      File f2 = new File(logDir, "aman.log");
      if (f2.exists()) {
          return f2;
      }
      File f3 = new File(logDir, "A-Man.log");
      if (f3.exists()) {
          return f3;
      }
      return f1;
  }
  ```
- `tailLog` 메소드에서 `File logFile = getActiveLogFile(logDir);`을 호출하여 조회하도록 수정합니다.
