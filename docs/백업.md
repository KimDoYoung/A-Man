# A-Man 백업 전략 및 구현 가이드 (Backup Strategy & Implementation Guide)

이 문서는 A-Man 시스템의 데이터 유실을 방지하기 위한 백업 방안과 이를 시스템 자체적으로 수행하기 위한 **스프링 부트 백업 스케줄러 & 관리자 페이지 다운로드 API** 구현 설계를 정리한 문서입니다.

---

## 1. 백업 대상 정의

백업이 필요한 데이터는 `aman.base-dir` 하위의 다음 두 가지 디렉토리 및 파일입니다.

1. **SQLite 3 데이터베이스 파일**:
   - 경로: `${aman.base-dir}/db/aman.db`
   - 주의: DB에 쓰기 트랜잭션이 활성화된 상태에서 단순 파일 복사(`cp`)를 하면 파일이 손상(corruption)될 수 있습니다. 반드시 SQLite의 `.backup` 명령어 또는 `VACUUM INTO` 쿼리를 사용해야 합니다.
2. **이미지 디렉토리**:
   - 경로: `${aman.base-dir}/data/images`
   - 내용: 도움말 웹에 삽입된 모든 정적 스크린샷 파일들.

## 2. 세 가지 백업 방안 요약

| 구분 | 방안 A: 웹 UI 원클릭 백업 다운로드 | 방안 B: 인프라 크론탭 배치 (Cron) | 방안 C: 스프링 부트 자체 백업 스케줄러 |
| :--- | :--- | :--- | :--- |
| **레벨** | 애플리케이션 REST API 및 화면 연동 | 리눅스 OS 레벨 스케립팅 | 애플리케이션 내부 백그라운드 스레드 |
| **동작 방식** | 관리자가 화면에서 버튼을 클릭하여 수동 다운로드 | cron 서비스가 주기적으로 쉘 스크립트 실행 | Spring `@Scheduled`가 지정된 시간에 작업 실행 |
| **저장 위치** | 관리자 로컬 PC (웹 브라우저 다운로드) | 서버 내 로컬 디렉토리 (`/backups`) | 서버 내 로컬 디렉토리 (`/backups`) |
| **최대 장점** | 비개발자/일반인 관리자가 손쉽게 백업 가능 | 서버 프로세스 다운과 무관하게 백업 가동 | 별도 OS 크론탭 설정 없이 패키지 구동만으로 해결 |

### 3.2 백그라운드 백업 비즈니스 로직 및 스케줄러

#### `BackupService.java` (신규 파일)
- 경로: `backend/src/main/java/kr/co/kfs/aman/service/BackupService.java`
- 역할: SQLite 안전 백업 생성, 이미지 디렉토리 병합 압축 및 오래된 백업 보존 관리 수행.
```java
package kr.co.kfs.aman.service;

import org.slf4j.Logger;
import org.slf4j.LoggerFactory;
import org.springframework.beans.factory.annotation.Value;
import org.springframework.stereotype.Service;
import org.springframework.transaction.annotation.Transactional;

import javax.persistence.EntityManager;
import javax.persistence.PersistenceContext;
import java.io.*;
import java.nio.file.Files;
import java.text.SimpleDateFormat;
import java.util.Date;
import java.util.zip.ZipEntry;
import java.util.zip.ZipOutputStream;

@Service
public class BackupService {

    private static final Logger log = LoggerFactory.getLogger(BackupService.class);

    @PersistenceContext
    private EntityManager entityManager;

    @Value("${aman.db-name}")
    private String dbPath;

    @Value("${aman.image-dir}")
    private String imageDir;

    @Value("${aman.backup.dir:${aman.base-dir}/backups}")
    private String backupDir;

    @Value("${aman.backup.retention-days:30}")
    private int retentionDays;

    /**
     * SQLite DB를 안전하게 백업한 후, 이미지 폴더와 함께 임시 ZIP 파일로 묶어 리턴합니다.
     */
    @Transactional
    public File createBackupZip() throws Exception {
        // 임시 작업용 디렉토리 생성
        File tempDir = Files.createTempDirectory("aman-backup-").toFile();
        File tempDbBackup = new File(tempDir, "aman.db");

        try {
            // SQLite 'VACUUM INTO' 구문 실행 (동시 쓰기 상황에서도 안전하게 읽기 스냅샷으로 복사본 생성)
            if (tempDbBackup.exists()) {
                tempDbBackup.delete();
            }
            entityManager.createNativeQuery("VACUUM INTO '" + tempDbBackup.getAbsolutePath() + "'").executeUpdate();
            log.info("SQLite DB vacuum-into 백업 완료: {}", tempDbBackup.getAbsolutePath());

            // 임시 ZIP 파일 생성
            File zipFile = File.createTempFile("aman_backup_temp_", ".zip");
            try (FileOutputStream fos = new FileOutputStream(zipFile);
                 ZipOutputStream zos = new ZipOutputStream(fos)) {
                
                // 1. DB 파일 압축 엔트리 추가
                addFileToZip(tempDbBackup, "db/aman.db", zos);
                
                // 2. 이미지 폴더 압축 엔트리 추가
                File imgDir = new File(imageDir);
                if (imgDir.exists() && imgDir.isDirectory()) {
                    addDirectoryToZip(imgDir, "data/images", zos);
                }
            }
            return zipFile;
        } finally {
            // 복사된 임시 db 파일 제거
            if (tempDbBackup.exists()) {
                tempDbBackup.delete();
            }
            tempDir.delete();
        }
    }

    /**
     * 정기 백업 태스크 실행 및 오래된 백업 정리
     */
    public void runScheduledBackup() {
        try {
            log.info("정기 백업 작업을 시작합니다.");
            File backupZip = createBackupZip();

            File destDir = new File(backupDir);
            if (!destDir.exists()) {
                destDir.mkdirs();
            }

            // 백업 완료 본 저장소로 이동
            String timestamp = new SimpleDateFormat("yyyyMMdd_HHmmss").format(new Date());
            File finalZip = new File(destDir, "aman_backup_" + timestamp + ".zip");
            Files.move(backupZip.toPath(), finalZip.toPath());
            log.info("정기 백업 완료 파일 저장 위치: {}", finalZip.getAbsolutePath());

            // 보존 기한이 지난 백업 정리
            purgeOldBackups(destDir);
        } catch (Exception e) {
            log.error("정기 백업 수행 중 에러 발생", e);
        }
    }

    private void purgeOldBackups(File destDir) {
        File[] files = destDir.listFiles((dir, name) -> name.startsWith("aman_backup_") && name.endsWith(".zip"));
        if (files == null) return;

        long cutoff = System.currentTimeMillis() - ((long) retentionDays * 24 * 60 * 60 * 1000);
        for (File f : files) {
            if (f.lastModified() < cutoff) {
                if (f.delete()) {
                    log.info("만료된 백업 파일 제거 완료: {}", f.getName());
                }
            }
        }
    }

    private void addFileToZip(File file, String entryName, ZipOutputStream zos) throws IOException {
        try (FileInputStream fis = new FileInputStream(file)) {
            ZipEntry zipEntry = new ZipEntry(entryName);
            zos.putNextEntry(zipEntry);
            byte[] bytes = new byte[1024];
            int length;
            while ((length = fis.read(bytes)) >= 0) {
                zos.write(bytes, 0, length);
            }
            zos.closeEntry();
        }
    }

    private void addDirectoryToZip(File folder, String parentFolder, ZipOutputStream zos) throws IOException {
        File[] files = folder.listFiles();
        if (files == null) return;
        for (File file : files) {
            if (file.isDirectory()) {
                addDirectoryToZip(file, parentFolder + "/" + file.getName(), zos);
                continue;
            }
            addFileToZip(file, parentFolder + "/" + file.getName(), zos);
        }
    }
}
```

#### `BackupScheduler.java` (신규 파일)
- 경로: `backend/src/main/java/kr/co/kfs/aman/scheduler/BackupScheduler.java`
- 역할: `application.properties`에 지정된 cron식 주기마다 정기 백업 호출.
```java
package kr.co.kfs.aman.scheduler;

import kr.co.kfs.aman.service.BackupService;
import org.slf4j.Logger;
import org.slf4j.LoggerFactory;
import org.springframework.beans.factory.annotation.Value;
import org.springframework.scheduling.annotation.Scheduled;
import org.springframework.stereotype.Component;

@Component
public class BackupScheduler {

    private static final Logger log = LoggerFactory.getLogger(BackupScheduler.class);

    private final BackupService backupService;

    @Value("${aman.backup.enabled:true}")
    private boolean enabled;

    public BackupScheduler(BackupService backupService) {
        this.backupService = backupService;
    }

    @Scheduled(cron = "${aman.backup.cron:0 0 2 * * ?}")
    public void executeBackupTask() {
        if (!enabled) {
            log.info("백업 스케줄러가 비활성화 상태입니다.");
            return;
        }
        backupService.runScheduledBackup();
    }
}
```

### 3.3 웹 UI 다운로드용 REST API 컨트롤러

#### `BackupController.java` (신규 파일)
- 경로: `backend/src/main/java/kr/co/kfs/aman/controller/BackupController.java`
- 역할: `/admin/backup/download` 엔드포인트를 제공하여 압축파일 스트리밍 및 전송 완료 후 임시파일 청소.
```java
package kr.co.kfs.aman.controller;

import kr.co.kfs.aman.service.BackupService;
import org.springframework.core.io.FileSystemResource;
import org.springframework.http.HttpHeaders;
import org.springframework.http.MediaType;
import org.springframework.http.ResponseEntity;
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.RestController;

import java.io.File;
import java.text.SimpleDateFormat;
import java.util.Date;

@RestController
@RequestMapping("/admin/backup")
public class BackupController {

    private final BackupService backupService;

    public BackupController(BackupService backupService) {
        this.backupService = backupService;
    }

    @GetMapping("/download")
    public ResponseEntity<?> downloadBackup() {
        try {
            File zipFile = backupService.createBackupZip();
            String timestamp = new SimpleDateFormat("yyyyMMdd_HHmmss").format(new Date());
            String fileName = "aman_backup_" + timestamp + ".zip";

            return ResponseEntity.ok()
                    .header(HttpHeaders.CONTENT_DISPOSITION, "attachment; filename=\"" + fileName + "\"")
                    .contentType(MediaType.parseMediaType("application/zip"))
                    .body(new FileSystemResource(zipFile) {
                        @Override
                        public File getFile() {
                            File file = super.getFile();
                            // JVM 및 요청 종료 후 임시 압축파일 삭제 보장 설정
                            file.deleteOnExit();
                            return file;
                        }
                    });
        } catch (Exception e) {
            return ResponseEntity.badRequest().body("백업 파일을 생성하는 데 실패했습니다: " + e.getMessage());
        }
    }
}
```

## 4. 방안 B: 서버 인프라 쉘 스크립트 + Cron 설정

서버 로컬 백업 자동화를 위한 쉘 스크립트 기반 가이드라인입니다.

#### `scripts/backup.sh` 예시
```bash
#!/usr/bin/env bash
# A-Man 백업 스크립트

set -euo pipefail

# 설정 (서버 환경에 맞추어 절대경로 지정 필요)
BASE_DIR="/data/docker/apps/aman"
BACKUP_DIR="${BASE_DIR}/backups"
RETENTION_DAYS=30

DATE_STR=$(date +%Y%m%d_%H%M%S)
TEMP_BACKUP="${BACKUP_DIR}/temp_${DATE_STR}"

mkdir -p "${BACKUP_DIR}"
mkdir -p "${TEMP_BACKUP}"

echo "[INFO] 백업 시작: ${DATE_STR}"

# 1. SQLite DB 안전 백업
if command -v sqlite3 &> /dev/null; then
    sqlite3 "${BASE_DIR}/db/aman.db" ".backup '${TEMP_BACKUP}/aman.db'"
    echo "[INFO] SQLite 안전 백업 완료"
else
    echo "[WARN] sqlite3 명령어가 없어 단순 복사를 시도합니다."
    cp "${BASE_DIR}/db/aman.db" "${TEMP_BACKUP}/aman.db"
fi

# 2. 이미지 디렉토리 복사
if [ -d "${BASE_DIR}/data/images" ]; then
    cp -r "${BASE_DIR}/data/images" "${TEMP_BACKUP}/images"
    echo "[INFO] 이미지 백업 완료"
else
    echo "[WARN] 이미지 디렉토리가 없습니다."
fi

# 3. 압축 파일 생성
tar -czf "${BACKUP_DIR}/aman_backup_${DATE_STR}.tar.gz" -C "${BACKUP_DIR}" "temp_${DATE_STR}"
rm -rf "${TEMP_BACKUP}"

echo "[INFO] 백업 파일 생성 완료: ${BACKUP_DIR}/aman_backup_${DATE_STR}.tar.gz"

# 4. 오래된 백업 파일 정리 (30일 보유 정책)
find "${BACKUP_DIR}" -name "aman_backup_*.tar.gz" -type f -mtime +${RETENTION_DAYS} -delete
echo "[INFO] ${RETENTION_DAYS}일 경과된 백업 파일 정리 완료."
```

## 5. 검증 및 복구 가이드

### 백업 유효성 검증
- 생성된 압축 파일의 크기가 0이 아닌지 체크합니다.
- 로컬 PC에서 압축을 해제하여 `aman.db` 파일이 있고 `images/` 디렉토리에 파일이 정상적으로 들어가 있는지 육안으로 확인합니다.
- 복사된 `aman.db` 파일을 SQLite 뷰어 도구(DBeaver, DB Browser for SQLite 등)로 열어서 데이터 조회가 정상적으로 수행되는지 검증합니다.

### 데이터 복구 절차
데이터베이스 손상이나 서버 이관 시 다음 절차에 따라 복구를 진행합니다.
1. A-Man WAS(Tomcat) 프로세스를 정지합니다.
2. 기존 `${aman.base-dir}/db/aman.db` 파일을 별도 이름으로 백업(예: `aman.db.bak`)하여 격리합니다.
3. 백업 파일의 `aman.db`를 `${aman.base-dir}/db/` 경로에 배치합니다.
4. 백업 파일의 `images/` 하위 파일들을 `${aman.base-dir}/data/images/` 경로에 덮어씁니다.
5. WAS(Tomcat) 프로세스를 기동하여 정상 조회 및 정상 수정 여부를 검증합니다.

## 백업전략

1. springboot를 이용한 스케줄러를 사용한다.
2. api /last_modify_time을 작성한다.
    - BackupController의 작성
    - pages, folders 테이블에서 최종 수정/생성 시각을 구한다.
    
3. 스케줄러
    - 오전 12:30, 오후 11:30 하루에 2번동작한다.
    - application-<profile>.properties에 기록한다.
        ```properties
        # Backup Settings
        aman.backup.enabled=true
        aman.backup.cron=0 0 2 * * ? <-수정
        aman.backup.dir=${aman.base-dir}/backup
        ```
    - 스케줄러에 의해서 2번 동작하는  backup process
    ```text
        - settings 테이블에서 LAST_BACKUP_TIME을 가져온다.
        - backup폴더에 *.war를 리스트해서 날짜순으로 10개만 유지, 10개가 넘으면 과거의 것들 삭제
        - backup폴더에  *.gz 를 리스트해서 날짜순으로 30개만 유지, 30개가 넘으면 과거의 것들 삭제
        - LAST_BACKUP_TIME이 /last_modify_time 보다 미래이면 동작하지 않는다. 즉 백업받고 변화한 것이 없다면 백업동작을 하지 않는다.
        - db를 백업해서 aman.yyyy-mm-dd_HH_MM_SS.db로 배업
        - images images-yyyy-mm-dd_HH_MM_SS.tar.gz으로 백업
        - 주의 yyyy-mm-dd_HH_MM_SS은 db와 images가 짝으로 동일해야한다.
         
    ```
4. 백업관리 : BackupPages.tsx를 작성.
    - 백업 directory에 있는 amaan-<ver>.war, db..gz, images.gz의 리스트를 보여준다.
    - 각각 다운로드 받을 수 있다.
