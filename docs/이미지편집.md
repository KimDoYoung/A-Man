#  이미지 편집

## 개요

- 이미지를 upload한 후에 **편집**해서 저장 또는 clipbard로 복사
- **편집**이란 원숫자(숫자넣기), 박스(사각형) 넣기, 문자(글씨) 넣기 이다. 

## 주요기능 

- 화면설명에 사용할 이미지( 이하 대상이미지)를 로드
- 대상이미지를 clipboard에서 붙여넣기
- 편집한 이미지(이하 편집이미지)를 다운로드(편집 이미지)
  (편집이미지란 대상이미지에 item들을 입힌 이미지이다)
- 대상이미지 위에 item들을 추가한다.
- item이란 원숫자, 박스, 문자 등이다.
    - 원숫자 : 사용자에게 번호를 붙여서 특정위치의 기능이나 설명을 하기 위해서 추가한다. 
    - 박스 : 특정영역을 표시하고 설명을 하기 위함 
    - 문자 : 문자를 넣어서 설명하기 위함
- 입히는 동작을 하나 하나 action이라고 한다.
- action리스트에서 삭제가 가능해서 undo기능을 수행한다.
- item들은 attributes 를 갖는다.
    - 원숫자 : 테두리 색깔, 테두리 라인두께, 바탕색깔, 글자색갈
    - 박스 : 박스 테두리 색깔, 테두리 라인두께
    - 문자 : 문자 색깔, 문자 크기, 문자를 감싸는 박스 추가 여부,

  ## 🛠️ 이미지 편집 및 보관 (ImageWork) 기능 상세 기술

  ### 1. UI/UX 디자인 철학 (Simplicity & Defaults)

  • 불필요한 기능 제거: 화려하고 복잡한 기능이 가득한 픽픽(PicPick)과 달리, 도움말 매뉴얼 제작자가 화면 가이드에 쓸
  필수 핵심 기능만 최전선에 노출합니다.
  • 센스 있는 기본값(Default) 제공: 사용자가 마우스 클릭을 할 때 매번 속성이나 스타일(색상, 굵기 등)을 튜닝하지 않아도
  되게끔 매뉴얼 표준 가이드용 기본값을 사전에 탑재해 둡니다. (예외적인 수정을 원할 때만 컨트롤 판넬을 건드리도록 설계)
  • 워크플로우 단순화: 클립보드 복사(Copy)와 스크린샷 붙여넣기(Paste) 연동을 핵심으로 삼아, 작업 편의성을 극대화합니다.

  ### 2. 에디터 제공 핵심 도구 (Essentials)

  1. 원숫자 마킹 (Stamp):
      • 동작: 클릭하는 위치에 번호가 적힌 원을 그립니다.
      • 기본값: 번호가 순차적으로 자동 증가 ( ①  ──>  ②  ──>  ③ ), Indigo 배경색에 흰색 텍스트.
  2. 강조 박스 (Box):
      • 동작: 화면의 특정 입력창이나 버튼 영역을 강조 표시하기 위해 사각형을 그립니다.
      • 기본값: Red 테두리, 선 두께  2px , 투명 배경.
  3. 텍스트 입력 (Text):
      • 동작: 간략한 캡션을 캔버스에 직접 기입합니다.
      • 기본값: 검은색( text-slate-900 ) 폰트, 크기  14px , 감싸는 배경 박스 없음.
  4. 점선 자르기 (Crop):
      • 동작: 스크린샷에서 필요한 영역만 고르는 고무줄(러버밴드) 점선 선택식 Crop 기능.

  ### 3. 데이터베이스 테이블 DDL ( image_work )

  스키마의 유연성을 위해 title, original_image_url, edited_image_url 및 개별 도형 데이터( items ) 전체를 단일 JSON
  텍스트 필드에 통째로 묶어 담는 방식을 채택했습니다.
```sql
    -- 이미지 편집 작업 상태를 단일 JSON 필드로 통합하여 관리하는 테이블 DDL
    CREATE TABLE image_work (
        id          INTEGER PRIMARY KEY AUTOINCREMENT,
        user_id     INTEGER NOT NULL,			-- 소유한 사용자 ID (외래키)
        title       TEXT NOT NULL,          -- json_data와 중복으로 보관        
        json_data   TEXT NOT NULL,			-- title, original_image_url, edited_image_url, items(도형들)를
  포함한 통합 JSON
        created_at  DATETIME DEFAULT CURRENT_TIMESTAMP, -- 생성 시각
        updated_at  DATETIME DEFAULT CURRENT_TIMESTAMP, -- 최종 수정 시각 (정렬용)

        FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
    );

    -- 사용자별 최신 수정일 기준 정렬 조회를 위한 복합 인덱스
    CREATE INDEX idx_image_work_user_updated ON image_work (user_id, updated_at DESC);
```

  ### 🛡️ 기존 소스 영향도 0%를 위한 격리 설계 방안

  #### 1. 데이터베이스(SQLite) 수준에서의 격리

  • 독립된 신규 테이블:  image_work  테이블은 기존  pages ,  folders ,  users  테이블의 스키마를 전혀 수정하지 않고
  완전히 새로 독립 구축됩니다.
  • 영향성 없음: 타 테이블의 DDL이나 관계 제약이 변경되지 않으므로, 기존 CRUD 작업의 무결성이 100% 보장됩니다.

  #### 2. 백엔드(Java Spring Boot) 수준에서의 격리

  • 독립 패키지 및 클래스: 신규 소스인  ImageWork.java (JPA 모델),  ImageWorkRepository.java ,  ImageWorkController.
  java 만 신설합니다.
  • 무영향: 기존  ContentController.java 나  FolderController.java  등 기존 자바 소스 파일은 단 한 줄의 import나 로직도
  건드리지 않습니다. 따라서 백엔드 빌드 및 컴파일 안정성이 완벽하게 격리됩니다.

  #### 3. 프론트엔드(React + TS) 수준에서의 격리

  • 독립 모듈 폴더화: 이미지 에디터 관련 코드는 전체가  src/components/imageditor/  폴더 하위에 전용 컴포넌트(Canvas
  제어 로직, UI 패널 등)로만 모여서 작성됩니다.
  • 모달(Modal) 오버레이 호출 방식:
      • 에디터 진입은 단순하게  DocUserTopBar.tsx 에 "이미지 편집" 버튼을 하나 추가하고, 이 버튼 클릭 시
      <ActionImageEditor isOpen={isOpen} onClose={...} /> 와 같은 독립형 모달 창만 화면 위에 띄우는 방식을 취합니다.
      • 기존 에디터 화면이나 전체 레이아웃 상태를 훼손하지 않기 때문에 프론트엔드 렌더링 런타임 오류가 기존 화면에
      전파될 염려가 전혀 없습니다.
