# 백틱 확장 파싱 테스트 문서 (bt_examples.md)

이 문서는 A-Man 백틱 파싱 확장의 검증을 위한 예제 파일입니다.

1. **풀 포맷 (색상 + 아이콘 + 텍스트)**
   - 노란색 배경에 편집 아이콘 등록 버튼: `ffff00:fa fa-edit:등록`

2. **색상 생략 (기본 색상 + 아이콘 + 텍스트)**
   - 기본 배경에 플러스 아이콘 추가 버튼: `:fas fa-plus:추가`

3. **아이콘 생략 (색상 + 텍스트)**
   - 붉은색 배경에 저장 텍스트: `ff0000::저장`
   - 노란색 배경에 인사정보 가져오기 텍스트: `ffff00::인사정보 가져오기`

4. **색상, 아이콘 모두 생략 (사전 등록 단어 자동 룩업)**
   - 기등록 단어 "등록" (초록색 배경 + edit 아이콘): `등록`
   - 기등록 단어 "삭제" (붉은색 배경 + trash 아이콘): `삭제`
   - 기등록 단어 "조회" (파란색 배경 + search 아이콘): `조회`

5. **텍스트 생략 (색상 + 아이콘)**
   - 기밀 및 배경색만 표현: `:fa fa-plus:`
   - 노란색 배경과 편집 아이콘 단독 표현: `ffff00:fa fa-edit:`

6. **예외 및 미매칭 형식 처리**
   - 형식이 맞지 않음 (콜론 1개): `:abc`
   - 미등록 단어 (일반 kbd 스타일): `임시저장`
   - 일반 코드 내의 텍스트가 파싱되면 안 됨: `const val = 10;`

7. **한 줄 내 여러 백틱 혼용**
   - 먼저 `조회` 버튼을 누르고, `등록`을 한 다음 최종적으로 `저장`을 완료하십시오.
